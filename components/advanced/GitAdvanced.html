Git Advanced


1. Git Rebasing

  • Git Rebasing is a process used in version control with Git to integrate changes from one branch into another.
  • It allows user to move or apply a series of commits from one branch onto another branch, typically resulting in a more linear and cleaner commit history.
  • Rebasing is useful for maintaining a clear history and resolving conflicts between branches.

How to Rebase ?

  1. Start with a base branch (e.g., master or main).
  2. Create a feature branch.
  3. Make your changes in the feature branch.
  4. Move the changes from the feature branch onto the latest commit in the base branch by using below command.

    5.     git rebase [base_branch]

Result : The commit history looks as if the changes in the feature branch were made directly on top of the base branch, creating a more straightforward timeline.


2. Git Hooks

  • Git hooks are custom scripts or programs that user can set up to be automatically executed at specific points in the Git version control process.
  • These scripts allow user to automate tasks, enforce coding standards, perform pre-commit checks, send notifications, and more.
  • There are two types of hooks present in Git.
    1. Client-Side Hooks
    2. Server-Side Hooks

Client-Side Hooks
  • pre commit, prepare-commit-msg, commit-msg, post-commit, post-checkout, pre-rebase.
Server-Side Hooks
  • pre-receive, update, post-receive.

How to use Git Hooks ?

  1. Navigate to Your Git Repository.
  2. Locate the Hooks Directory: cd /path/to/your/repo/.git/hooks
  3. Choose or Create a Hook Script.
  4. Write Your Hook Script: To create a custom pre-commit hook, you can create a file named pre-commit in the hooks directory
  5. Customize Your Script.
  6. Testing (Optional).
  7. Commit Your Changes:
    8.     git add .git/hooks/pre-commit
    git commit -m "Add custom pre-commit hook"
  8. Finally you can use the Hook.
